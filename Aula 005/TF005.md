# TF - Tarefa Final - Aula 5

### Disciplina: Implementação de servidor e nuvem (cloud)
### Aula: 5 - Redes no Docker
### Objetivo do TF: Avaliar a compreensão dos drivers de rede, a importância das Redes Customizadas e a proficiência na configuração e *troubleshooting* da comunicação entre contêineres.

### Orientações:

1.  Responda às questões teóricas com base no conteúdo da aula.
2.  Execute os comandos práticos no seu terminal **WSL/Linux**.
3.  As respostas devem ser objetivas e baseadas nos comandos práticos.

### Questão 1: Redes Customizadas e DNS Interno (Teórica)
Um desenvolvedor lança um contêiner de *frontend* e um contêiner de *backend* separadamente, e ambos se conectam à **rede `bridge` padrão** do Docker. O *frontend* não consegue se comunicar com o *backend* usando o nome do contêiner.

a) **Explique** por que a comunicação por nome (DNS Interno) falha na rede `bridge` padrão.
b) Qual é a **melhor prática** para resolver esse problema e qual comando Docker você usaria para criar essa rede?

### Questão 2: Mapeamento de Portas vs. Comunicação Interna (Prática e Teórica)
Um contêiner Nginx está configurado para ouvir na porta 80.

a) **Prática:** Rode um contêiner Nginx (`nginx:latest`) dando a ele o nome `web_server_tf5`, e mapeando a porta 80 do contêiner para a porta **8888** do Host.
    *(Liste o comando `docker run` completo utilizado).*

b) **Teórica:** Qual é a finalidade principal do *port mapping* (`-p`) neste cenário? Ele é usado para comunicação **interna** entre o `web_server_tf5` e um banco de dados, ou para **acesso externo**?

### Questão 3: Conectividade Interna (Prática e Troubleshooting)
Você precisa provar que dois contêineres conseguem se comunicar usando o DNS Interno.

a) **Prática:** Execute os comandos necessários para:
    1. Criar uma rede customizada chamada `rede_app_tf5`.
    2. Rodar um contêiner (`nginx:latest`) chamado `servico_a` **conectado a esta rede**.
    3. Rodar um segundo contêiner (`alpine`) chamado `servico_b` **conectado a esta mesma rede**.
    4. **Entrar no terminal** do `servico_b` e usar um comando Linux para **testar a conectividade** com o `servico_a`.
    *(Liste o comando de criação da rede, os dois comandos `docker run` e o comando Linux de teste de conectividade).*

b) **Comando de Verificação:** Qual comando Docker você usaria para visualizar o **endereço IP** que o Docker atribuiu ao contêiner `servico_a` dentro da rede `rede_app_tf5`? (Foque no comando de inspeção de rede).

### Questão 4: Limpeza e `ip a` (Prática e Teórica)
Após completar os testes na Questão 3, você precisa limpar o ambiente.

a) **Prática:** Usando o comando `docker exec -it [contêiner] [comando]`, entre no contêiner `servico_b` e execute o comando Linux para **visualizar todas as interfaces de rede** do contêiner.
    *(Liste o comando `docker exec` completo e o comando Linux utilizado).*

b) **Teórica:** Qual sequência de comandos é necessária para garantir a **remoção completa** da rede customizada `rede_app_tf5`? (Assuma que todos os contêineres foram parados e removidos).

### Questão 5: Comandos de Inspeção (Teórica)
Qual comando do Docker CLI é mais adequado para obter uma saída detalhada no formato JSON sobre a configuração da sua rede customizada `rede_app_tf5`, incluindo a sub-rede, o *gateway* e a lista de contêineres que estão anexados a ela?
