### Disciplina: Implementação de servidor e nuvem (cloud)
### Aula: 8 - Docker Compose e Ambientes Multi-Contêiner
### Objetivo do TF: Avaliar a proficiência na definição de serviços, volumes e redes usando a sintaxe YAML do Docker Compose para orquestrar um ambiente de aplicação (Web) e persistência (Database).

## Questão 1: Conceito Central do Compose (Teórica)

## Questão 2: Estrutura YAML e Volumes (Prática Teórica)
Você precisa definir um *stack* com um serviço de banco de dados PostgreSQL.

a) **Prática:** Crie o trecho YAML completo para o serviço `db_postgres` que atenda aos seguintes requisitos:
    1. Use a imagem `postgres:15-alpine`.
    2. Defina uma variável de ambiente `POSTGRES_USER` com o valor `user_app`.
    3. Use um Volume Nomeado chamado `pg_data` para persistir o diretório de dados padrão (`/var/lib/postgresql/data`).

b) Qual é o comando Docker Compose que você deve usar para garantir que os **logs** de todos os serviços (incluindo o `db_postgres`) sejam exibidos em tempo real no seu terminal?

## Questão 3: Build, Rede e Dependência (Prática Teórica)
Seu serviço *Web* (`app_web`) usa um `Dockerfile` localizado no diretório `./app/` e depende do `db_postgres` para iniciar.

a) **Prática:** Crie o trecho YAML para o serviço `app_web` que atenda aos seguintes requisitos:
    1. **Construa** a imagem a partir do contexto `./app/`.
    2. Defina o mapeamento de portas: Host `5000` para Contêiner `80`.
    3. Use a seção `depends_on` para garantir que o `db_postgres` inicie antes.

b) Qual é o **nome de Host** que o código da aplicação (dentro de `app_web`) deve usar para se conectar ao `db_postgres`? (Foque na regra de DNS Interno do Compose).
