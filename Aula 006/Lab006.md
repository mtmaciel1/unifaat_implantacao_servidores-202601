# Lab 6: Docker Compose e Ambientes Multi-Contêiner

**Disciplina:** Implementação de servidor e nuvem (cloud)  
**Aula:** 6 - Docker Compose e Ambientes Multi-Contêiner  
**Objetivo:** Criar o primeiro `docker-compose.yml` para orquestrar uma aplicação Web (que exige *build* local) e um Database (que exige persistência por Volume), provando o DNS Interno do Compose.

## Objetivo do Lab

Ao final deste laboratório, você será capaz de:
- Criar e estruturar arquivos docker-compose.yml
- Orquestrar múltiplos serviços com um único comando
- Integrar build de imagens customizadas no Compose
- Configurar volumes e redes automaticamente
- Utilizar DNS interno entre serviços
- Gerenciar dependências entre contêineres

## Pré-requisitos

* Docker Desktop rodando e integrado ao WSL/Linux.
* Conhecimento de Dockerfile e Volumes.
* **Ferramenta:** `docker-compose` instalado (ou `docker compose` para versões mais recentes).

## Cenário: Orquestração de uma Aplicação Web + Database

Vamos criar um *stack* com dois serviços interconectados usando o `docker-compose.yml`.

### Seção 1: Preparação do Ambiente e Estrutura de Build

O Compose precisa de um diretório de contexto (`./app`) para construir a imagem do frontend.

1.  Crie o diretório raiz do projeto (`project_compose_8`).
```bash
mkdir ~/aulas_lab/aula008
cd ~/aulas_lab/aula008
```
2.  Crie o subdiretório para o *build* do Frontend (`app`) e o arquivo principal de orquestração (`docker-compose.yml`).
```bash
mkdir app
touch docker-compose.yml
```

3. Inicialise o VSCode
```bash
code .
```

4.  Crie um `Dockerfile` simples dentro da pasta `app/`.
```dockerfile
FROM nginx:alpine
COPY index.html /usr/share/nginx/html/index.html
EXPOSE 80
```
4.  Crie um arquivo HTML( app/index.html ) para ser copiado.
```html
<h1>AULA 8: Compose Funcionando!</h1>
```

### Seção 2: Criação do `docker-compose.yml`

O arquivo YAML deve integrar as configurações de Build, Rede, Volumes e Variáveis de Ambiente. 

**Crie o arquivo docker-compose.yml Copie e cole o seguinte conteúdo para ele:**

```yaml
services:
  # Serviço 1: Frontend (App Web)
  web_app:
    build: 
      context: ./app  # Instrução para construir a partir do Dockerfile em ./app
      dockerfile: Dockerfile
    container_name: compose-web-lab8
    ports:
      # Mapeamento para o acesso externo (Host 8000 -> Contêiner 80)
      - "8000:80"
    environment:
      # Variável para referência interna (DNS Interno)
      - DB_HOST=db_cache
    depends_on:
      - db_cache # Garante que o DB inicie primeiro

  # Serviço 2: Database (Redis)
  db_cache:
    image: redis:alpine
    container_name: compose-redis-lab8
    volumes:
      # Monta o volume nomeado para persistência de dados
      - redis_data_lab8:/data
    # Nenhuma porta mapeada para o Host, pois será acessado apenas internamente

volumes:
  # Definição explícita do volume nomeado
  redis_data_lab8:
    driver: local
```

### Seção 3: Execução e Validação do Stack

Com o arquivo `docker-compose.yml` e o `Dockerfile` prontos, podemos lançar todo o ambiente com um único comando.

1.  **Lance o Stack:** Use o comando para construir e iniciar os serviços em modo *detached* (`-d`).

```bash
docker-compose up -d --build
```

*Observe que o Compose faz automaticamente: 
- Build da `web_app`. 
- Criação do volume `redis_data_lab8`. 
- Criação da rede customizada `project_compose_8_default`. 
- Início dos contêineres.*

2.  **Verificação dos Serviços:** Confirme que os dois contêineres estão ativos.

 ```bash
 docker-compose ps
 ```

3.  **Verificação Externa:** Teste o acesso ao serviço Web pela porta mapeada (`8000`).
```bash
curl localhost:8000
```

*A saída deve ser o conteúdo do seu `index.html`, confirmando o *build* e o mapeamento de porta.*

### Seção 4: Prova da Comunicação Interna (DNS)

Vamos provar que o serviço `web_app` consegue se comunicar com o `db_cache` usando o **nome do serviço** (`db_cache`).

1.  Entre no terminal Bash do contêiner `web_app`:

```bash
docker-compose exec web_app sh
```

2.  **Teste de Conectividade:** Dentro do contêiner, use o `ping` para testar o nome do serviço do Redis:

```bash
ping db_cache
```

> **Resultado Esperado:** O `ping` deve ter sucesso, confirmando que o Compose ativou o DNS Interno.

3.  Saia do contêiner:

```bash
exit
```

### Seção 5: Limpeza Completa

O comando de limpeza do Compose é poderoso e fundamental.

1.  **Derrube e Remova Tudo:** Use a *flag* `-v` para garantir que até mesmo o Volume Nomeado persistente seja removido, limpando totalmente o sistema.

```bash
docker-compose down -v
```

> Este comando remove contêineres, redes e o volume `redis_data_lab8`.)*

2.  **Limpeza Final (Opcional):** Remova o diretório do projeto:

```bash
cd ~
rm -rf ~/aulas_lab/aula008
```

## Dicas Importantes

1. **Use docker-compose.yml** para ambientes de desenvolvimento e testes
2. **Organize serviços** com nomes descritivos e dependências claras
3. **Sempre use volumes nomeados** para persistência de dados
4. **Aproveite o DNS interno** para comunicação entre serviços
5. **Limpe com -v** para remover volumes ao derrubar o stack
