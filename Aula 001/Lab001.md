# Aula 1 — Fundamentos, Contêineres e Setup do Ambiente WSL/Linux

**Disciplina:** Implementação de servidor e nuvem (cloud)<br>
**Aula:** 1 - Fundamentos, Contêineres e Setup do Ambiente WSL/Linux<br>
**Objetivo do Lab:** Estabelecer um ambiente Linux funcional no Windows (WSL2) e praticar comandos essenciais de administração Linux para navegação e manipulação de arquivos.

**Observação:** Este guia foca na configuração e verificação do WSL em Windows e na execução dos comandos dentro da distribuição Linux (ex.: Ubuntu). Execute comandos do Windows (PowerShell) quando indicado; o restante dos comandos deve ser executado no terminal da distribuição Linux (WSL).

## Seção 1: Setup e Verificação do WSL

1. Verificar se o WSL está instalado e em qual versão (no PowerShell com privilégios de administrador):

```powershell
wsl -l -v
```

Saída esperada (exemplo):

```
  NAME      STATE           VERSION
* Ubuntu    Running         2
```

- Se aparecer `VERSION` como `2` para a sua distro (ex.: `Ubuntu`), o WSL2 está ativo.
- Se o comando não for reconhecido ou aparecer versão `1`, instale/atualize o WSL com (PowerShell em modo administrador):

```powershell
wsl --install -d Ubuntu
# ou para atualizar para WSL2
wsl --set-default-version 2
```

2. Iniciar o terminal da distribuição (Ubuntu):

- Pelo PowerShell/Prompt de Comando:

```powershell
wsl
```

Após entrar, você estará em um shell Linux dentro do WSL (prompt similar a `usuario@DESKTOP:~$`).

## Seção 2: Comandos de Navegação e Arquivos (Prática Dirigida)

Crie um diretório de prática e execute os seguintes passos. Execute todos os comandos no shell da distribuição (Ubuntu/WSL).

1. Quem você é no sistema:
```bash
whoami
# Ex.: aletavares
```
2. Executar comandos com privilégios de superusuário (root). Essencial para instalação de pacotes:
```bash
sudo whoami
# Ex.: root
```

3. Confirmar diretório atual com `pwd`:

```bash
pwd
# Ex.: /home/seu_usuario
```
4. Atualizar a lista de pacotes do linux:
```bash
sudo apt update
```
5. Criar um diretório para o lab e entrar nele (`mkdir`, `cd`):

```bash
mkdir -p ~/aulas_lab/aula001
cd ~/aulas_lab/aula001
```
> O parâmetro **-p (parents)** permite criar diretórios pais que não existem
> **Exemplo:** se ~/aulas_lab não existir, ele será criado automaticamente antes de criar aula001
> Sem o -p, o comando falharia se o diretório pai não existisse


6. Criar subdiretórios e arquivos de exemplo (`mkdir`, `touch`):

```bash
mkdir docs temp
touch README.txt nota1.txt nota2.txt
```

7. Listar o conteúdo (`ls`) e ver detalhes (`ls -l`):

```bash
ls
ls -l
```

8. Mover/renomear um arquivo (`mv`):

```bash
mv nota1.txt docs/nota1.txt
ls docs
```

9. Criar um arquivo temporário dentro de `temp`, depois remover recursivamente (`rm -r`):

```bash
touch temp/tmpfile.txt
ls temp
rm -r temp
# Confirmar remoção
ls
```

10. Todos os comandos exigidos nesta seção:

| Comando | Descrição |
|---------|-----------|
| pwd            | Mostra o diretório atual (Print Working Dir)     |
| mkdir -p       | Cria diretórios e subdiretórios se necessário    |
| cd             | Muda para o diretório especificado               |
| touch          | Cria arquivos vazios ou atualiza timestamp       |
| ls -l          | Lista arquivos com detalhes (permissões/datas)   |
| mv             | Move/renomeia arquivos e diretórios              |
| rm -r          | Remove diretórios e seu conteúdo recursivamente  |

## Seção 3: Permissões e Administração

1. Criar um script shell simulado:
- Abra o VSCode na pasta
```bash
code .
```
- crie um arquivo chamado script_simulado.sh com o conteúdo abaixo:
```bash
#!/bin/bash
echo "Iniciando script simulado..."
echo "Executando tarefa 1" 
sleep 1
echo "Executando tarefa 2"
sleep 1
echo "OK" > resultado.txt
echo "Script finalizado"
```

# Verificar conteúdo
```bash
cat script_simulado.sh
```

2. Dar permissão de execução com `chmod` e executar:

```bash
./script_simulado.sh
# ao tentar executar o comando vai dar erro de permissão de execução

chmod +x script_simulado.sh
./script_simulado.sh
```
> **EXPLICANDO:**<br  >
> **chmod:** comando para modificar permissões de arquivos/diretórios
> **+x:** adiciona (+) a permissão de execução (x) para todos os usuários
> **script_simulado.sh:** arquivo que receberá a permissão de execução

3. Uso de `echo` e operadores de redirecionamento `>` e `>>` para criar/atualizar um arquivo de log:

```bash
echo "$(date +'%F %T') - INICIO do lab" > lab.log
echo "Comando: ls -l" >> lab.log
ls -l >> lab.log
tail -n 20 lab.log
```

> **'>':** cria/ sobrescreve o arquivo. <br>
> **'>>':** acrescenta (append).


## Seção 4: Troubleshooting Básico (Verificação)**

1. Usar `grep` para filtrar informação específica. Exemplos:

- Filtrar linhas com `-` (arquivos regulares) de `ls -l`:

```bash
ls -l | grep "^-"
```

- Procurar a palavra `OK` no arquivo `resultado.txt` ou no log:

```bash
grep "OK" resultado.txt || echo "Não encontrado: OK"
grep "INICIO" lab.log
```

2. Se um comando falhar, verifique mensagens de erro e permissões. Exemplos comuns:

- Erro `Permission denied` ao executar um script: faltou `chmod +x`.
- Erro `No such file or directory`: caminho incorreto ou diretório não criado.

## Comandos de Verificação (Checklist final)

Execute os comandos abaixo para confirmar que o Lab foi concluído com sucesso. Deve haver saídas sem erros e arquivos/linhas esperadas:
1. Verificar execução do WSL (no Windows):
No PowerShell:
```bash
wsl -l -v
```

2. Verificar diretório da prática e arquivos
```bash
pwd
ls -la ~/aulas_lab/aula001
```
3. Verificar que o script foi criado, tem permissão e resultado
```bash
# Lista detalhada (-l) dos arquivos especificados, mostrando permissões, tamanho, data, etc
ls -l script_simulado.sh resultado.txt lab.log

# Executa o script redirecionando:
./script_simulado.sh >/dev/null 2>&1 || true
# '>/dev/null' - redireciona saída padrão para /dev/null (descarta)
# 2>&1 - redireciona stderr para stdout (também descarta erros)
# || true - retorna true mesmo se o comando falhar (não interrompe execução)

# Procura a string "OK" no arquivo resultado.txt
grep "OK" resultado.txt && echo "Resultado OK encontrado"
# && echo - só executa o echo se o grep encontrar "OK" (retornar true)

# Verificar log criado e última entrada
tail -n 5 lab.log
# O comando 'tail' mostra as últimas linhas de um arquivo
# -n 5: especifica para mostrar apenas as 5 últimas linhas
# lab.log: é o arquivo que será lido

# 5. Teste rápido de busca com grep
ls -l | grep "^-" && echo "Arquivos listados"
# Este comando é composto de 3 partes:
# 1. ls -l : Lista arquivos em formato longo, mostrando permissões, tamanho, data etc
# 2. grep "^-" : Filtra apenas linhas que começam com "-" (arquivos regulares)
#    ^ significa "início da linha" e - indica arquivo regular nas permissões
# 3. && echo "Arquivos listados" : Exibe a mensagem apenas se o comando anterior for bem sucedido

```
> Se todos os comandos acima rodarem sem erros (ou com mensagens esperadas), o ambiente está pronto e os comandos fundamentais foram praticados com sucesso.

## Dicas Rápidas

- Para abrir um terminal WSL rapidamente, use `wsl` no PowerShell ou CMD.
- Se precisar acessar arquivos do Windows dentro do WSL, eles ficam montados em `/mnt/c/` (Ex.: `cd /mnt/c/Users/SeuUsuario`).
- Para atualizar a distro Ubuntu: `sudo apt update && sudo apt upgrade -y`.

## Limpeza do Ambiente

Após concluir o laboratório, remova os arquivos e diretórios criados:

```bash
# Voltar para o diretório home
cd ~

# Remover todo o diretório de prática
rm -rf ~/aulas_lab/aula001
# Remove recursivamente (-r) e forçadamente (-f) todo o diretório aula001 e seu conteúdo
# -r: remove recursivamente (diretório e subdiretórios)
# -f: força a remoção sem pedir confirmação
# ~/aulas_lab/aula001: caminho do diretório a ser removido

# Verificar se foi removido
ls ~/aulas_lab/
```

**Importante:** O comando `rm -rf` remove recursivamente (-r) e força (-f) a remoção sem confirmação. Use com cuidado!
