## 1. Conceitos Fundamentais

### O que é Containerização?

A **containerização** é uma tecnologia de virtualização que permite empacotar uma aplicação junto com todas as suas dependências (bibliotecas, configurações, arquivos) em uma unidade portável chamada **container**. É como criar uma "caixa" que contém tudo que sua aplicação precisa para funcionar, garantindo que ela rode da mesma forma em qualquer ambiente.

Imagine que você desenvolveu um jogo no seu computador. Para que seus amigos joguem, eles precisariam instalar as mesmas versões de bibliotecas, drivers e configurações que você tem. Com containerização, você "empacota" o jogo com tudo que ele precisa, e seus amigos podem simplesmente "abrir a caixa" e jogar, independente do computador que tenham.

### Evolução da Virtualização: Físico → VM → Container

#### Servidores Físicos (Era Antiga)
Antigamente, cada aplicação precisava de um servidor físico dedicado:
- **Desperdício de recursos**: Servidor usado apenas 10-20% da capacidade
- **Custo alto**: Muitos servidores físicos
- **Inflexibilidade**: Difícil de escalar ou mover aplicações

#### Máquinas Virtuais (VMs)
As VMs revolucionaram a infraestrutura permitindo múltiplos "computadores virtuais" em um servidor físico:
- **Melhor aproveitamento**: Um servidor físico roda várias VMs
- **Isolamento**: Cada VM é independente
- **Problema**: Cada VM precisa de um Sistema Operacional completo (pesado)

#### Containers (Era Atual)
Os containers levaram a virtualização ao próximo nível:
- **Compartilhamento do OS**: Todos os containers usam o mesmo kernel do host
- **Leveza extrema**: Iniciam em milissegundos, ocupam megabytes
- **Portabilidade**: "Funciona no meu computador" = "Funciona em qualquer lugar"

### WSL: Ponte entre Windows e Linux

#### Por que WSL é Importante?
A maioria dos servidores em produção roda **Linux**, mas muitos desenvolvedores usam **Windows**. O WSL (Windows Subsystem for Linux) resolve esse problema permitindo rodar um ambiente Linux real dentro do Windows.

**Vantagens do WSL 2:**
- **Kernel Linux real**: Performance nativa para containers
- **Integração com Windows**: Acessa arquivos do Windows facilmente
- **Ferramentas Linux**: Bash, grep, sed, awk funcionam nativamente
- **Docker nativo**: Roda containers sem virtualização adicional

#### WSL vs Dual Boot vs VM
| Aspecto | WSL | Dual Boot | VM Tradicional |
|---------|-----|-----------|----------------|
| **Performance** | Nativa | Nativa | Reduzida |
| **Facilidade** | Alta | Baixa | Média |
| **Integração** | Excelente | Nenhuma | Limitada |
| **Recursos** | Compartilhados | Dedicados | Compartilhados |

### Comandos Linux Essenciais

#### Navegação e Estrutura
```bash
pwd                 # Mostra diretório atual
ls -la              # Lista arquivos com detalhes e ocultos
cd /caminho         # Navega para diretório
cd ..               # Volta um nível
cd ~                # Vai para home do usuário
```

#### Manipulação de Arquivos
```bash
mkdir -p pasta/subpasta    # Cria diretórios recursivamente
touch arquivo.txt          # Cria arquivo vazio
cp origem destino          # Copia arquivos
mv origem destino          # Move/renomeia arquivos
rm -rf pasta/              # Remove recursivamente (CUIDADO!)
```

#### Permissões e Execução
```bash
chmod +x script.sh         # Torna arquivo executável
chmod 755 arquivo          # Permissões específicas (rwxr-xr-x)
./script.sh                # Executa script no diretório atual
sudo comando               # Executa com privilégios de administrador
```

#### Redirecionamento e Pipes
```bash
comando > arquivo.txt      # Redireciona saída (sobrescreve)
comando >> arquivo.txt     # Redireciona saída (anexa)
comando | grep "texto"     # Pipe: saída de um vira entrada de outro
comando 2> erro.log        # Redireciona erros para arquivo
```

### Docker: A Revolução dos Containers

#### Arquitetura Docker
- **Docker Engine**: O "motor" que gerencia containers
- **Images**: Templates imutáveis para criar containers
- **Containers**: Instâncias em execução de uma image
- **Dockerfile**: Receita para construir images
- **Registry**: Repositório de images (Docker Hub, ECR)

#### Por que Docker Mudou Tudo?
1. **"Funciona na minha máquina"**: Problema resolvido com containers
2. **Deployment consistente**: Mesmo container em dev, teste e produção
3. **Escalabilidade**: Fácil criar/destruir containers conforme demanda
4. **Microserviços**: Cada serviço em seu próprio container
5. **DevOps**: Desenvolvedores e operações falam a mesma "linguagem"

## 2. Questões de Múltipla Escolha

### Questão 1
Qual é a principal vantagem dos containers em relação às máquinas virtuais tradicionais?

a) Containers incluem seu próprio sistema operacional, garantindo total isolamento
b) Containers compartilham o kernel do host, resultando em menor overhead e inicialização mais rápida
c) Containers só funcionam em Windows, enquanto VMs só funcionam em Linux
d) Containers são mais seguros porque cada um tem seu próprio hypervisor

**Gabarito: B**

**Justificativa:** A principal vantagem dos containers é o compartilhamento do kernel do sistema operacional host, eliminando a necessidade de carregar um OS completo para cada instância. Isso resulta em muito menor uso de recursos (CPU, RAM, disco) e inicialização quase instantânea comparado às VMs que precisam "bootar" um OS completo.

### Questão 2
No contexto do WSL (Windows Subsystem for Linux), qual é a principal razão para um desenvolvedor utilizá-lo em vez de uma máquina virtual tradicional?

a) WSL oferece melhor performance gráfica para jogos
b) WSL permite integração nativa entre ferramentas Linux e o sistema de arquivos Windows
c) WSL é gratuito enquanto VMs são pagas
d) WSL só funciona com containers Docker

**Gabarito: B**

**Justificativa:** O WSL foi projetado especificamente para integração entre Windows e Linux. Você pode acessar arquivos do Windows diretamente do terminal Linux, usar ferramentas Linux em projetos Windows, e vice-versa. Essa integração nativa é impossível com VMs tradicionais que são completamente isoladas.

### Questão 3
Você precisa criar um script que adicione informações de log a um arquivo existente sem apagar o conteúdo anterior. Qual operador de redirecionamento você deve usar?

a) > (maior que)
b) >> (duplo maior que)
c) < (menor que)
d) | (pipe)

**Gabarito: B**

**Justificativa:** O operador `>>` (append) adiciona conteúdo ao final do arquivo sem apagar o que já existe. O operador `>` sobrescreve o arquivo completamente, `<` é para entrada (input), e `|` é para conectar a saída de um comando à entrada de outro.
