# Lab 2: Arquitetura do Docker e Docker Desktop no WSL

**Disciplina:** Implementação de servidor e nuvem (cloud)  
**Aula:** 2 - Arquitetura do Docker e Docker Desktop no WSL  
**Objetivo:** Configurar o Docker Desktop com WSL, resolver problemas de permissão e dominar o gerenciamento de imagens e contêineres no terminal.

## Objetivo do Lab

Ao final deste laboratório, você será capaz de:
- Configurar e verificar a comunicação entre Cliente Docker (WSL) e Daemon (Docker Desktop)
- Resolver problemas de permissão do Docker sem usar `sudo`
- Executar o ciclo completo de vida de contêineres
- Gerenciar imagens Docker localmente
- Realizar troubleshooting básico em contêineres

## Pré-requisitos

- WSL2 instalado e configurado
- Docker Desktop instalado no Windows
- Terminal WSL aberto

## Seção 1: Verificação da Arquitetura e Permissões

### 1.1 Teste Inicial de Comunicação

Primeiro, vamos verificar se o Docker Desktop está se comunicando corretamente com o WSL:

```bash
docker run hello-world
```

**Saída esperada:**
```
Hello from Docker!
This message shows that your installation appears to be working correctly.
```

### 1.2 Resolução do Problema de Permissão

Se você recebeu um erro de permissão como `permission denied while trying to connect to the Docker daemon socket`, execute:

```bash
sudo usermod -aG docker $USER
```

**IMPORTANTE:** Após executar este comando, você deve:
1. Fazer logout do WSL: `exit`
2. Fechar o terminal WSL
3. Reabrir o terminal WSL
4. Testar novamente: `docker run hello-world`

### 1.3 Verificação da Versão e Status

```bash
# Verificar versão do Docker
docker --version

# Verificar informações do sistema Docker
docker info
```

## Seção 2: Gerenciamento de Imagens

### 2.1 Baixando uma Imagem

Vamos baixar a imagem Alpine Linux (uma distribuição muito leve):

```bash
docker pull alpine
```

**Saída esperada:**
```
Using default tag: latest
latest: Pulling from library/alpine
...
Status: Downloaded newer image for alpine:latest
```

### 2.2 Listando Imagens Locais

```bash
docker images
```

**Saída esperada:**
```
REPOSITORY    TAG       IMAGE ID       CREATED        SIZE
alpine        latest    c059bfaa849c   2 weeks ago    5.59MB
hello-world   latest    feb5d9fea6a5   16 months ago  13.3kB
```

### 2.3 Removendo uma Imagem

```bash
# liste os container 
docker ps -a

# pegue o nome do container do hello-world
docker rm <nome_do_container>

# Remover a imagem hello-world
docker rmi hello-world

# Verificar se foi removida
docker images
```

## Seção 3: Ciclo de Vida do Contêiner

### 3.1 Executando um Contêiner Nginx

Vamos criar um contêiner Nginx em modo detached (segundo plano):

```bash
docker run -d --name meu-nginx -p 8080:80 nginx
```

**Explicação dos parâmetros:**
- `-d`: Executa em modo detached (segundo plano)
- `--name meu-nginx`: Define um nome específico para o contêiner
- `-p 8080:80`: Mapeia a porta 8080 do host para a porta 80 do contêiner

### 3.2 Listando Contêineres Ativos

```bash
docker ps
```

**Saída esperada:**
```
CONTAINER ID   IMAGE     COMMAND                  CREATED         STATUS         PORTS                  NAMES
abc123def456   nginx     "/docker-entrypoint.…"   2 minutes ago   Up 2 minutes   0.0.0.0:8080->80/tcp   meu-nginx
```

### 3.3 Visualizando Logs em Tempo Real

```bash
docker logs -f meu-nginx
```

**Para sair do modo de acompanhamento de logs:** Pressione `Ctrl + C`

### 3.4 Testando o Nginx

Abra outro terminal WSL e teste:

```bash
curl http://localhost:8080
```

Ou acesse `http://localhost:8080` no seu navegador Windows.

## Seção 4: Inspeção e Limpeza (Troubleshooting)

### 4.1 Entrando no Contêiner para Troubleshooting

```bash
docker exec -it meu-nginx /bin/bash
```

**Dentro do contêiner, você pode:**
```bash
# Instalar Utilitarios do Linux no container
apt update && apt install -y procps

# Listar processos
ps aux

# Verificar arquivos de configuração do Nginx
ls -la /etc/nginx/

# Verificar logs internos
cat /var/log/nginx/access.log

# Sair do contêiner
exit
```

### 4.2 Inspecionando Detalhes do Contêiner

```bash
# Informações detalhadas do contêiner
docker inspect meu-nginx

# Verificar uso de recursos
docker stats meu-nginx --no-stream
```

### 4.3 Ciclo de Parada e Remoção

```bash
# Parar o contêiner
docker stop meu-nginx

# Verificar status (deve aparecer como "Exited")
    docker ps -a

# Remover o contêiner
docker rm meu-nginx

# Verificar se foi removido
docker ps -a
```

### 4.4 Limpeza Completa do Sistema

```bash
# Remover todos os contêineres parados, redes não utilizadas, imagens órfãs e cache
docker system prune

# Para uma limpeza mais agressiva (incluindo imagens não utilizadas)
docker system prune -a
```

## Comandos de Verificação - Resumo

### Comandos Essenciais Aprendidos:

| Comando | Descrição |
|---------|-----------|
| `docker run -d --name <nome> <imagem>` | Executa contêiner em segundo plano |
| `docker ps` | Lista contêineres ativos |
| `docker ps -a` | Lista todos os contêineres |
| `docker logs -f <contêiner>` | Acompanha logs em tempo real |
| `docker exec -it <contêiner> /bin/bash` | Entra no contêiner interativamente |
| `docker stop <contêiner>` | Para um contêiner |
| `docker rm <contêiner>` | Remove um contêiner |
| `docker images` | Lista imagens locais |
| `docker pull <imagem>` | Baixa uma imagem |
| `docker rmi <imagem>` | Remove uma imagem |
| `docker system prune` | Limpa recursos não utilizados |
| `docker inspect <contêiner>` | Mostra detalhes do contêiner |
| `docker stats <contêiner>` | Mostra uso de recursos |

### Fluxo Completo de Trabalho:

```bash
# 1. Baixar imagem
docker pull nginx

# 2. Executar contêiner
docker run -d --name web-server -p 8080:80 nginx

# 3. Verificar execução
docker ps

# 4. Acompanhar logs
docker logs -f web-server

# 5. Troubleshooting (se necessário)
docker exec -it web-server /bin/bash

# 6. Parar e limpar
docker stop web-server
docker rm web-server
docker system prune
```
