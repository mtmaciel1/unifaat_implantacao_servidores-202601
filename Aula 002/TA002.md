## 1. Conceitos Fundamentais

### Arquitetura Docker: Client-Daemon-Registry

O **Docker** funciona com uma arquitetura **cliente-servidor** que revolucionou como desenvolvemos e implantamos aplicações. É importante entender cada componente para usar o Docker efetivamente.

#### Docker Client (Cliente)
O **Docker Client** é a interface que você usa para interagir com o Docker. Quando você digita comandos como `docker run` ou `docker build`, está usando o cliente Docker. É como o "controle remoto" do Docker.

**Características:**
- Interface de linha de comando (CLI)
- Envia comandos para o Docker Daemon via API REST
- Pode estar na mesma máquina ou conectar remotamente
- Traduz comandos humanos em instruções para o daemon

#### Docker Daemon (Servidor)
O **Docker Daemon** é o "cérebro" do Docker. É um processo que roda em background no sistema operacional e faz todo o trabalho pesado: criar containers, gerenciar imagens, configurar redes.

**Responsabilidades:**
- Gerenciar ciclo de vida dos containers
- Construir imagens a partir de Dockerfiles
- Gerenciar volumes e redes
- Comunicar com registries para baixar/enviar imagens
- Expor API REST para clientes

#### Docker Registry (Repositório)
O **Docker Registry** é onde as imagens Docker são armazenadas e distribuídas. É como uma "biblioteca" de imagens que podem ser baixadas e usadas.

**Tipos de Registry:**
- **Docker Hub**: Registry público oficial (gratuito com limitações)
- **Amazon ECR**: Registry privado da AWS
- **Registry privado**: Hospedado pela própria empresa
- **Registry local**: Para desenvolvimento e testes

### Imagens vs Containers: A Diferença Fundamental

#### Docker Images (Imagens)
Uma **imagem Docker** é um template **imutável** que contém tudo necessário para rodar uma aplicação: código, runtime, bibliotecas, variáveis de ambiente e configurações.

**Características das Imagens:**
- **Imutáveis**: Uma vez criada, não muda
- **Camadas (Layers)**: Construídas em camadas para eficiência
- **Versionadas**: Identificadas por tags (ex: `nginx:1.21`, `ubuntu:20.04`)
- **Reutilizáveis**: Uma imagem pode gerar múltiplos containers

**Analogia:** A imagem é como uma "receita de bolo". A receita não muda, mas você pode fazer vários bolos (containers) a partir dela.

#### Docker Containers (Contêineres)
Um **container** é uma **instância em execução** de uma imagem Docker. É o "bolo feito" a partir da "receita" (imagem).

**Características dos Containers:**
- **Mutáveis**: Podem ser modificados durante execução
- **Efêmeros**: Podem ser criados e destruídos facilmente
- **Isolados**: Cada container tem seu próprio ambiente
- **Stateful**: Podem manter estado durante execução (mas é perdido ao parar)

### Comandos Docker Essenciais

#### Gerenciamento de Imagens
```bash
docker images                    # Lista imagens locais
docker pull ubuntu:20.04        # Baixa imagem do registry
docker rmi ubuntu:20.04         # Remove imagem local
docker build -t minha-app .     # Constrói imagem a partir de Dockerfile
docker history ubuntu:20.04     # Mostra histórico de camadas
```

#### Gerenciamento de Containers
```bash
docker run ubuntu:20.04         # Cria e executa container
docker run -it ubuntu bash      # Executa interativamente com terminal
docker run -d nginx             # Executa em background (detached)
docker ps                       # Lista containers rodando
docker ps -a                    # Lista todos containers (incluindo parados)
docker stop container_id        # Para container
docker start container_id       # Inicia container parado
docker rm container_id          # Remove container
```

#### Comandos de Inspeção
```bash
docker logs container_id        # Mostra logs do container
docker exec -it container_id bash  # Executa comando dentro do container
docker inspect container_id     # Mostra detalhes completos do container
docker stats                    # Mostra uso de recursos em tempo real
```

### Ciclo de Vida dos Containers

#### Estados do Container
1. **Created**: Container criado mas não iniciado
2. **Running**: Container executando ativamente
3. **Paused**: Container pausado (processo suspenso)
4. **Stopped**: Container parado (processo finalizado)
5. **Deleted**: Container removido do sistema

#### Transições de Estado
```
Created → Running (docker start)
Running → Paused (docker pause)
Paused → Running (docker unpause)
Running → Stopped (docker stop)
Stopped → Running (docker start)
Any State → Deleted (docker rm)
```

### Docker Desktop: Integração com WSL

#### Vantagens do Docker Desktop
- **Interface gráfica**: Facilita gerenciamento para iniciantes
- **Integração WSL2**: Performance nativa no Windows
- **Kubernetes integrado**: Para aprendizado de orquestração
- **Atualizações automáticas**: Mantém Docker sempre atualizado

#### Configurações Importantes
- **WSL 2 Backend**: Usar sempre para melhor performance
- **Resource Limits**: Configurar CPU e memória adequadamente
- **File Sharing**: Permitir acesso aos diretórios de projeto
- **Experimental Features**: Habilitar para recursos avançados

## 2. Questões de Múltipla Escolha

### Questão 1
Na arquitetura Docker, qual é a função principal do Docker Daemon?

a) Fornecer interface de linha de comando para o usuário
b) Armazenar e distribuir imagens Docker pela internet
c) Gerenciar containers, imagens, volumes e redes no sistema operacional host
d) Criar interfaces gráficas para facilitar o uso do Docker

**Gabarito: C**

**Justificativa:** O Docker Daemon é o componente central que executa no sistema operacional host e é responsável por todo o gerenciamento de containers, imagens, volumes e redes. Ele recebe comandos do Docker Client via API REST e executa as operações solicitadas. As outras opções descrevem o Docker Client (A), Docker Registry (B) e Docker Desktop (D).

### Questão 2
Qual é a diferença fundamental entre uma imagem Docker e um container Docker?

a) Imagem é para Linux e container é para Windows
b) Imagem é um template imutável, container é uma instância executável da imagem
c) Imagem ocupa mais espaço em disco que container
d) Não há diferença, são sinônimos

**Gabarito: B**

**Justificativa:** A imagem Docker é um template imutável (não muda) que contém tudo necessário para rodar uma aplicação. O container é uma instância em execução dessa imagem, que pode ser modificada durante a execução. É como a diferença entre uma "receita" (imagem) e o "prato preparado" (container). Uma imagem pode gerar múltiplos containers.

### Questão 3
Você executou o comando `docker run -d nginx` e quer verificar se o container está rodando corretamente. Qual comando você deve usar?

a) `docker images`
b) `docker ps`
c) `docker pull nginx`
d) `docker build nginx`

**Gabarito: B**

**Justificativa:** O comando `docker ps` lista todos os containers que estão atualmente em execução, mostrando informações como ID, imagem usada, comando executado, status e portas. O parâmetro `-d` no `docker run` executa o container em background (detached), então você precisa do `docker ps` para verificar se está rodando. As outras opções servem para: listar imagens (A), baixar imagens (C) e construir imagens (D).
