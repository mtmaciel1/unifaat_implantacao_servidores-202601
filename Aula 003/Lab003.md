## Seção 2: Criação do Dockerfile Base

### 2.1 Escrevendo o Dockerfile Inicial

```Dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "app.py"]
```

**Explicação das instruções:**
- `FROM`: Define a imagem base (Python 3.9 slim)
- `WORKDIR`: Define o diretório de trabalho dentro do contêiner
- `COPY`: Copia arquivos do host para o contêiner
- `RUN`: Executa comandos durante o build
- `CMD`: Define o comando padrão ao executar o contêiner

## Seção 5: Otimização Avançada (Encadeamento de RUN)

### 5.1 Dockerfile Não Otimizado (Estado Atual)

Nosso Dockerfile atual tem comandos RUN separados. Vamos criar uma versão com múltiplos RUNs para demonstrar:

```Dockerfile
# Criar Dockerfile com múltiplos RUNs
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt

RUN apt-get update

RUN apt-get install -y curl && apt-get clean

COPY . .

CMD ["python", "app.py"]
```

### 5.2 Build da Versão Não Otimizada

```bash
# Build da versão não otimizada
docker build -t minha-app:nao-otimizada .

# Analisar camadas
docker history minha-app:nao-otimizada
```

**Observe:** Cada comando RUN cria uma nova camada!

### 5.3 Dockerfile Otimizado (Encadeamento Obrigatório)

Agora vamos **encadear os comandos RUN** em uma única instrução:

```Dockerfile
# Criar Dockerfile otimizado com RUN encadeado
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .

RUN pip install --no-cache-dir -r requirements.txt && \
    apt-get update && \
    apt-get install -y curl && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

COPY . .

CMD ["python", "app.py"]
```

### 5.4 Build da Versão Otimizada

```bash
# Build da versão otimizada
docker build -t minha-app:otimizada .
```

### 5.5 Comparação CRUCIAL das Camadas

```bash
# Comparar camadas - VERSÃO NÃO OTIMIZADA
echo "=== VERSÃO NÃO OTIMIZADA ==="
docker history minha-app:nao-otimizada --format "table {{.ID}}\t{{.CreatedBy}}\t{{.Size}}"

echo ""
echo "=== VERSÃO OTIMIZADA ==="
# Comparar camadas - VERSÃO OTIMIZADA
docker history minha-app:otimizada --format "table {{.ID}}\t{{.CreatedBy}}\t{{.Size}}"
```

**Resultado esperado:**
- **Não otimizada:** 3 camadas RUN separadas
- **Otimizada:** 1 camada RUN única

### 5.6 Comparação de Tamanhos

```bash
# Comparar tamanhos finais
docker images | grep minha-app
```

## Análise dos Resultados

### Diferenças Observadas:

| Versão | Número de Camadas RUN | Tamanho | Cache Efficiency |
|--------|----------------------|---------|------------------|
| Não Otimizada | 3 camadas separadas | Maior | Menor |
| Otimizada | 1 camada única | Menor | Maior |

### Vantagens do Encadeamento:
- **Menos camadas** = imagem mais eficiente
- **Melhor cache** = builds mais rápidos
- **Menor tamanho** = menos espaço em disco
- **Limpeza automática** = sem arquivos temporários

## Comandos de Verificação - Resumo

### Comandos Essenciais Aprendidos:

| Comando | Descrição |
|---------|-----------|
| `docker build -t <nome:tag> .` | Constrói imagem a partir do Dockerfile |
| `docker history <imagem>` | **CHAVE:** Lista camadas da imagem |
| `docker images` | Lista imagens locais |
| `docker run --rm <imagem>` | Executa contêiner e remove após parar |

### Técnicas de Otimização:

#### 1. **Arquivo .dockerignore**
```bash
# Exemplo de .dockerignore
debug.log
*.tmp
.git
__pycache__
```

#### 2. **Encadeamento de RUN (Técnica Principal)**
```dockerfile
# NÃO OTIMIZADO (múltiplas camadas)
RUN apt-get update
RUN apt-get install -y curl
RUN apt-get clean

# OTIMIZADO (uma camada)
RUN apt-get update && \
    apt-get install -y curl && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*
```

#### 3. **Ordem das Instruções**
```dockerfile
# ORDEM OTIMIZADA
FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .          # Arquivo que muda menos
RUN pip install -r requirements.txt
COPY . .                         # Código que muda mais
CMD ["python", "app.py"]
```

### Fluxo Completo de Otimização:

```bash
# 1. Criar contexto
mkdir projeto && cd projeto

# 2. Criar .dockerignore
echo "*.log" > .dockerignore

# 3. Escrever Dockerfile otimizado
cat > Dockerfile << 'EOF'
FROM python:3.9-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt && \
    apt-get update && \
    apt-get install -y curl && \
    apt-get clean
COPY . .
CMD ["python", "app.py"]
EOF

# 4. Build e análise
docker build -t app:otimizada .
docker history app:otimizada
```

## Dicas Importantes

1. **Sempre use .dockerignore** para excluir arquivos desnecessários
2. **Encadeie comandos RUN** para reduzir camadas
3. **Ordene instruções** das menos mutáveis para as mais mutáveis
4. **Use docker history** para analisar o impacto de cada instrução
5. **Limpe arquivos temporários** no mesmo RUN que os cria
